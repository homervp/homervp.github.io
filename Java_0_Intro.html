Introduction to Java

Java is a cross-platform language created by James Gosling in Sun Microsystems in the early 90’s and first published in 1995 to meet great success with the public. Sun has since been purchased by Oracle, which continues evolving Java, having recently released Java 8.

Java does not compile directly to an executable, but rather to bytecode which is executed through the Java Virtual Machine (JVM). This means that as long as there is a JVM release for the specific system in use, there is no need to recompile or port a project to that system, but the same compilation can be ran directly on the system.

Java is generally said to be a sibling to C++, however it is apparent and well established that it takes a different approach and has equal similarities with other languages. Java features fewer low level commands, while it’s code is managed, meaning that in contrast to C and C++, and similarly to C#, it does not give the user access to memory management, not allowing pointers, memory allocation and other low level features, but rather handles memory automatically. Though this may occasionally make implementing something you would normally do with pointers rather hard, it does make it harder to have rogue pointers which may change wrong data, and theoretically makes Java applications safer towards the machine.
